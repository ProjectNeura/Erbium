Support guidance

These notes cover the most common join and leave issues without requiring a separate runbook.

Before you join

Confirm you only request the GPUs you need and set a realistic runtime window.

  • Shorter runtime requests reduce total wait time for the rest of the queue.
  • Available GPU badges are informational; your job is still placed into the scheduler queue.
  • Use a memorable job name so you can identify and remove the request later if plans change.

If a join request fails

The form validates obvious input issues first, then shows API errors inline.

  • Check that runtime is a positive number of hours.
  • Make sure at least one GPU is selected.
  • Retry after refresh if live scheduler data has not loaded yet.

If a leave request fails

A leave action only succeeds when the queued job is found by exact credentials.

  • Use the original job name exactly as submitted.
  • Use the same SSH password stored with that request.
  • If the job has already started running, it is no longer in the queue and cannot be removed here.
